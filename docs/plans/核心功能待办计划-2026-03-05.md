# 核心功能待办计划（2026-03-05）

## 1. 目标与范围

本计划聚焦以下 5 项核心功能的落地与补齐：

1. 智能筛选策略（基于技术指标和基本面的多维筛选）
2. 评分系统（可配置评分机制）
3. 自选股管理（分组 + 备注）
4. 筛选会话（保存与复用筛选结果）
5. 历史数据分析（基于历史数据的指标计算）

当前阶段判断：

- 领域模型、tRPC 路由、Prisma 持久化主链路已搭建
- 可运行但未达到“稳定可交付”
- 优先目标：先补齐数据与执行链路正确性，再完善前端可用性

## 3. 分功能待办清单

## 3.1 智能筛选策略

### 现状

- 已支持 `AND/OR/NOT` 递归条件组
- 已支持 BASIC/TIME_SERIES/DERIVED 指标分类
- 已有策略 CRUD 与执行入口

### 待办

- [ ] 扩展技术指标枚举与元数据（MA、EMA、RSI、MACD、KDJ、布林带等）
- [ ] 为技术指标补充数据来源与计算实现（Python 服务 + TS 映射）
- [ ] 增加“指标方向”配置（越大越好/越小越好）并接入筛选和评分
- [ ] 为复杂条件增加可视化表达结构（供前端构建器复用）
- [ ] 为筛选执行增加“预估命中数”快速接口（不执行完整评分）

### 验收标准

1. 策略可同时包含基本面 + 技术面条件并正确执行
2. 条件嵌套 3 层以上可稳定保存、读取、执行
3. 单次筛选无类型错误、无 Promise 泄漏、结果可复现

---

## 3.2 评分系统

### 现状

- 已支持权重配置、权重校验、MIN_MAX 归一化、加权求和

### 待办

- [x] 修复 `ScreeningStrategy.execute` 与 `ScoringService` 的同步/异步契约不一致
- [x] 在 `ScoringConfig` 增加每指标评分方向（ASC/DESC）（已完成：2026-03-06）
- [x] 增加归一化策略扩展点（如 Z_SCORE，保持向后兼容）（已完成：2026-03-06）
- [x] 对文本型指标在评分配置层做前置校验（禁止进入数值评分）（已完成：2026-03-06）
- [x] 增加评分解释字段（每只股票输出贡献项说明）（已完成：2026-03-06）

### 验收标准

1. 评分执行无类型断言绕过、无异步竞态
2. 低值更优指标（如 PE）排序方向正确
3. 输出包含总分 + 分项贡献 + 原始指标值

- [x] 增加分页与排序能力（按创建时间/更新时间/股票数）（已完成：2026-03-06）
- [x] 增加防重复标签与标签规范化（大小写、空白）（已完成：2026-03-06）
- [ ] 前端完成列表页、详情页、标签筛选页

### 验收标准
